# Bead bf-4351: Analyze which Pluck settings hide beads

**Completed:** 2026-07-09

## Summary

Analyzed all Pluck configuration settings to identify which configurations could cause beads to be hidden from selection during NEEDLE's Pluck strand evaluation.

## Key Findings

### Settings That Hide Beads:

1. **`exclude_labels`** (Primary configurable hiding mechanism)
   - Location: `.needle.yaml` → `strands.pluck.exclude_labels`
   - Any bead with a label in this array is completely excluded
   - Current ARMOR setting: `[]` (empty - no exclusions)

2. **Status Filtering** (Built-in, not configurable)
   - Only beads with `status: open` are visible
   - Beads with `status: closed`, `status: in_progress`, `status: deleted` are hidden

3. **Assignee Filtering** (Built-in, not configurable)
   - Beads assigned to another worker are hidden from other workers
   - Only unassigned beads or beads assigned to the current worker are visible

### Settings That Do NOT Hide Beads:

1. **`sort_order`** - Only affects order, not visibility
2. **`split_after_failures`** - Creates new beads, doesn't hide existing ones
3. **Debug logging (`RUST_LOG`)** - Only affects output verbosity

### Combined Filtering Effects:

Filters are applied in sequence:
1. Label exclusion (`exclude_labels`)
2. Status filtering (closed/in_progress/deleted)
3. Assignee filtering (assigned to other workers)
4. Sorting (does not filter)

A bead must pass ALL filters to be visible.
