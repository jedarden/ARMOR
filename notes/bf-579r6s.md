2. **ClusterIP:**
   - S3 API: `http://10.21.118.151:9000`
   - Admin API: `http://10.21.118.151:9001`

3. **Short form (within armor namespace):**
   - S3 API: `http://armor:9000`
   - Admin API: `http://armor:9001`

### External Access

The ARMOR service is currently configured as ClusterIP only and is **NOT directly accessible from outside the cluster**. Options for external access:

1. **kubectl port-forward** (temporary development/testing):
   ```bash
   kubectl port-forward -n armor svc/armor 9000:9000 9001:9001
   ```
   Then access via `http://localhost:9000` or `http://localhost:9001`

2. **Ingress/NodePort** (not currently configured):
   Would require modifying the service type or adding Ingress resources

3. **VPN access to cluster network:**
   If connected to the cluster network via VPN, may be able to access pod/node IPs directly
