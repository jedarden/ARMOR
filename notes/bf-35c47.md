## Correct Approach: Test Through Public Interface

Instead of directly accessing internal fields, tests should use the public interface:

- `err.Error()` - Verify formatted message contains expected values
- `err.Code()` - Verify error code
- `err.YAMLErrorType()` - Verify error type
- `err.Context()` - Verify context string

The task description referenced "undefined field references (filePath0, filePath1, etc.)" but no such patterns exist in the codebase. The actual issue is **direct field access to internal implementation details**, which:
