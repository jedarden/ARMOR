## Three YAML Multi-Line Scalar Styles

### 1. Literal Style (`|`)
Preserves all newlines and formatting. Used for code blocks, scripts, etc.

```yaml
script: |
  #!/bin/bash
  # This is a comment in the script (literal content)
  echo "value #hash"
  url=http://example.com#anchor
```

**Key behavior:** All content including `#` symbols is preserved as literal text.

### 2. Folded Style (`>`)
Folds newlines into spaces (except blank lines, which are preserved as line breaks). Used for long text descriptions.

```yaml
description: >
  This is a folded text line
  # This looks like comment but is content
  Value: # Another hash
  All lines get folded into one
```

**Key behavior:** Newlines are folded into spaces, but `#` symbols in content are preserved.

### 3. Quoted Scalars (`"` or `'`)
Can span multiple lines. All content including `#` is preserved.

```yaml
text: "multi-line
quoted scalar with # hash symbols"
```

**Key behavior:** `#` symbols inside quotes are ALWAYS content, never comments.

## Comment Detection Rules

1. **Start of line comment:** `#` at start of line (after optional whitespace)
   - `"# comment"` → `LineType::Comment`

2. **Inline comment:** `#` preceded by whitespace within content
   - `"key: value # comment"` → strips to `"key: value "`

3. **Hash not preceded by whitespace:** Preserved as content
   - `"url:http://example.com#anchor"` → `#` preserved
   - `"color:#FFFFFF"` → `#` preserved

4. **Hash inside quoted scalars:** ALWAYS preserved as content
   - `"color: \"#FFFFFF\""` → `#` preserved
   - `"text: 'value # hash'"` → `#` preserved

### Pattern 2: Hash Preservation
```rust
#[test]
fn test_hash_preservation() {
    let test_cases = vec![
        "url: \"http://example.com#anchor\"",
        "color: \"#FFFFFF\"",
    ];
    
    for line in test_cases {
        assert!(!is_comment_line(line));
        let stripped = strip_inline_comment(line);
        assert!(stripped.contains('#'), "Hash should be preserved in: {}", line);
    }
}
```

### Pattern 3: Inline Comment Stripping
```rust
#[test]
fn test_inline_comment_stripping() {
    let test_cases = vec![
        ("key: value # comment", "key: value "),
        ("text: \"#hash\" # inline", "text: \"#hash\" "),
    ];
    
    for (line, expected) in test_cases {
        assert!(!is_comment_line(line));
        let stripped = strip_inline_comment(line);
        assert_eq!(stripped, expected);
    }
}
```

## Key Insight: Block Scalar Context

**Important limitation:** The current line-based parser doesn't track block scalar context statefully. This means:

- Lines inside `|` or `>` blocks that start with `#` are classified as `LineType::Comment` by `classify_line_type()`
- This is **expected behavior** since the parser analyzes lines independently
- A full YAML parser would track block context and preserve these as content

**Example:**
```yaml
script: |
  # This is literal content (NOT a comment in full YAML)
```

Current behavior: `"  # This is literal content"` → `LineType::Comment` (line parser limitation)

Full YAML parser behavior: This line would be tracked as literal block content.
