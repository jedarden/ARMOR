# Bead bf-2778z: LITESTREAM_ACCESS_KEY_ID Retrieval Attempt

## Task
Retrieve and decode LITESTREAM_ACCESS_KEY_ID from the armor-writer secret in ord-devimprint cluster.

## Finding
**BLOCKED by RBAC** - The read-only kubectl-proxy on ord-devimprint (accessed via `http://kubectl-proxy-ord-devimprint:8001`) runs with the `devpod-observer` ServiceAccount which has RBAC that explicitly denies secret access.

### Command Attempted
```bash
kubectl --server=http://kubectl-proxy-ord-devimprint:8001 \
  get secret armor-writer -n devimprint \
  -o jsonpath='{.data.LITESTREAM_ACCESS_KEY_ID}' | base64 -d
```

### Error Result
```
Error from server (Forbidden): secrets "armor-writer" is forbidden: 
User "system:serviceaccount:devpod-observer:devpod-observer" cannot get 
resource "secrets" in API group "" in the namespace "devimprint"
```

## Context
This is an expected blocker documented in the genesis bead. The ord-devimprint cluster is accessed only via a read-only proxy that does not have secret permissions. Alternative approaches would be:
1. Request secret read permissions for devpod-observer SA (not recommended for security)
2. Use direct kubeconfig with elevated permissions (if available)
3. Have an admin with cluster access retrieve the value through a secure channel

## Status
Bead complete - blocker confirmed and documented.
