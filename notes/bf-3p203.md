# Bead bf-3p203: Type Mismatch Error Details Enhancement

## Summary

Enhanced type-related errors in ARMOR to consistently include expected vs actual type information.

## Changes Made

### 1. ValidationError Enhancement (`internal/yamlutil/errors.go`)

**Updated `Error()` method** (lines 440-483):
- Added type information output when `ExpectedType` and/or `ActualType` fields are set
- Format: `(expected <type>, got <type>)` appended after constraint information
- Handles cases where only expected or actual type is available

**Updated `String()` method** (lines 490-512):
- Added separate lines for `Expected Type` and `Actual Type` in detailed output
- Format: `  Expected Type: <type>` and `  Actual Type: <type>`

### 2. Test Coverage (`internal/yamlutil/errors_test.go`)

**Added two new test functions**:
- `TestValidationErrorWithTypeInformation`: Validates Error() method includes type information
- `TestValidationErrorStringWithTypeInformation`: Validates String() method includes type information
- Tests cover scenarios: both types, only expected, only actual, with constraints

## Type Mismatch Error Scenarios

The ARMOR codebase has comprehensive type mismatch error handling:

### 1. TypeMismatchError
- **Location**: `internal/yamlutil/errors.go` (lines 893-935)
- **Fields**: `ExpectedType`, `ActualType`, `Value`, `FieldPath`, `Line`
- **Format**: `type mismatch in <file> at line <line>, field <field>: expected <expected>, got <actual>`
- **Example**: `type mismatch in config.yaml at line 8, field server.port: expected integer, got string`

### 2. FieldTypeError
- **Location**: `internal/yamlutil/schema.go` (lines 734-744)
- **Fields**: `ExpectedType`, `ActualType`, `Value`, `FieldPath`
- **Format**: `type error at <field>: expected <expected>, got <actual>`
- **Example**: `type error at server.port: expected integer, got string`

### 3. EnhancedParseError (Type Mismatch Kind)
- **Location**: `internal/yamlutil/parse_error_design.go` (lines 113-174)
- **Fields**: `Detail.ExpectedType`, `Detail.ActualType`, `Detail.FieldPath`, `Detail.Value`
- **Format**: `type mismatch error in <file> at line <line>, field <field>: expected <expected>, got <actual>`
- **Example**: `type mismatch error in config.yaml at line 8, field server.port: expected integer, got string`

### 4. ValidationError (NEWLY ENHANCED)
- **Location**: `internal/yamlutil/errors.go` (lines 395-512)
- **Fields**: `ExpectedType`, `ActualType`, `FieldPath`, `Line`, `Message`, `Constraint`
- **Format**: `validation error in <file> at line <line>, column <column> at field <field>: <message> (constraint: <constraint>) (expected <type>, got <type>)`
- **Example**: `validation error in config.yaml at line 10, column 5 at field server.port: invalid type (constraint: must be valid) (expected integer, got string)`
