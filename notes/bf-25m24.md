## Findings

### Issue: Direct Field Access on `ParseError` Struct

The test file contains **14 instances** of direct field access to `ParseError` struct fields, bypassing encapsulation. This violates the principle that tests should use public interfaces rather than internal implementation details.

### Current State of `ParseError` (from errors.go)

```go
type ParseError struct {
    FilePath   string      // Public field
    Line       int         // Public field
    Column     int         // Public field
    Message    string      // Public field
    ContextStr string      // Public field
    Err        error       // Public field
    ErrorType  ErrorType   // Public field
    ErrorCode  ErrorCode   // Public field
    Expected   string      // Public field
    Actual     string      // Public field
}
```

### Available Accessor Methods

- `Code() ErrorCode` - Returns error code
- `YAMLErrorType() ErrorType` - Returns error type
- `Context() string` - Returns context
- `Error() string` - Returns full error message
- `Unwrap() error` - Returns underlying wrapped error

### Missing Accessors

**No accessors exist for:**
- `Message` - Primary error message (distinct from full Error())
- `Line` - Line number
- `Column` - Column number
- `FilePath` - File path
- `Expected` - Expected value (for syntax errors)
- `Actual` - Actual value found (for syntax errors)

### Phase 2: Add Helper Methods for Safe Mutation

**File:** `/home/coding/ARMOR/internal/yamlutil/errors.go`

Add methods for creating modified copies (immutable pattern):

```go
// WithMessage returns a new ParseError with the message modified
func (pe *ParseError) WithMessage(msg string) *ParseError {
    copy := *pe
    copy.Message = msg
    return &copy
}
```
