## Root Cause Analysis

The `ParseError` struct in `errors.go` exposes its fields directly without providing accessor methods:

```go
type ParseError struct {
	FilePath   string
	Line       int
	Column     int
	Message    string
	ContextStr string
	Err        error
	ErrorType  ErrorType
	ErrorCode  ErrorCode
	Expected   string
	Actual     string
}
```

### Currently Available Methods on ParseError

- `Code() ErrorCode` ✓
- `YAMLErrorType() ErrorType` ✓
- `Context() string` ✓ (returns ContextStr)
- `Error() string` ✓
- `Unwrap() error` ✓ (returns Err)

### Missing Accessor Methods

- `GetMessage() string` or similar
- `GetLine() int`
- `GetColumn() int`

## Fix Plan

### Two-Phase Approach

#### Phase 1: Fix Field Mutation (Critical - 3 instances)

Fix the 3 instances of direct field mutation that violate error construction patterns:

**Issue 1: Direct Message field modification (Lines 139, 152)**

```go
// Current (WRONG):
mapped := result.MapErr(func(e *ParseError) *ParseError {
    e.Message = "modified: " + e.Message
    return e
})

// Fix Option A: Use constructor
mapped := result.MapErr(func(e *ParseError) *ParseError {
    return NewParseError(
        e.FilePath,
        "modified: " + e.Message,
        e.Line,
        e.Column,
        e.ErrorCode,
        e.Expected,
        e.Actual,
        e.ContextStr,
    )
})

// Fix Option B: Add WithMessage() method
func (pe *ParseError) WithMessage(newMsg string) *ParseError {
    copy := *pe
    copy.Message = newMsg
    return &copy
}

mapped := result.MapErr(func(e *ParseError) *ParseError {
    return e.WithMessage("modified: " + e.Message)
})
```

**Issue 2: Direct Err field assignment (Line 523)**

```go
// Current (WRONG):
parseErr := NewParseError("", fmt.Sprintf("invalid number: %s", s), 0, 0, "", "", "")
parseErr.Err = err

// Fix Option A: Extend NewParseError signature
// Change signature to include err parameter

// Fix Option B: Add WrapError() method
func (pe *ParseError) WrapError(wrapped error) *ParseError {
    copy := *pe
    copy.Err = wrapped
    return &copy
}

parseErr := NewParseError("", fmt.Sprintf("invalid number: %s", s), 0, 0, "", "", "").WrapError(err)

// Fix Option C: Remove error wrapping from example (least intrusive)
parseErr := NewParseError("", fmt.Sprintf("invalid number: %s", s), 0, 0, "", "", "")
```

**Recommendation for Phase 1:**
- Use Option B for MapErr (add `WithMessage()` method)
- Use Option B for ExampleResult (add `WrapError()` method)
- These follow immutable error patterns and are cleanest for tests

#### Phase 2: Add Accessor Methods (Optional - addresses remaining 12 read accesses)

Add public accessor methods to `ParseError` for safe field access:

```go
// GetMessage returns the error message.
func (pe *ParseError) GetMessage() string {
	return pe.Message
}

// GetLine returns the line number where the error occurred.
func (pe *ParseError) GetLine() int {
	return pe.Line
}

// GetColumn returns the column number where the error occurred.

	return pe.Column
}
```

Then update test assertions:
- `result.UnwrapErr().Message` → `result.UnwrapErr().GetMessage()`
- `converted.Line != 5` → `converted.GetLine() != 5`
- etc.

**Note:** Phase 2 is lower priority since field reading for test verification is acceptable. The critical issue is field mutation.
