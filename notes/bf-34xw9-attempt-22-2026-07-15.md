## Why External Restore Cannot Work

The litestream restore command requires:

```bash
litestream restore -config litestream.yml databases/queue.db
```

With configuration:
```yaml

  - path: databases/queue.db
    replica:
      type: s3
      endpoint: http://100.80.255.8:9000  # ← Unreachable
      access-key-id: ${LITESTREAM_ACCESS_KEY_ID}     # ← Available
      secret-access-key: ${LITESTREAM_SECRET_ACCESS_KEY}  # ← MISSING
```

**Two fundamental blockers:**
1. No SECRET_ACCESS_KEY → Cannot authenticate to ARMOR S3 proxy
2. No endpoint connectivity → Cannot reach ARMOR even with credentials
