## Acceptance Criteria Met

✅ **Create auth error test table extending the base structure**
- Created `create_auth_test_table()` function in `tests/test_tables.py`
- Extends the base `ErrorTestTable` and `TestCase` structures
- Table name: `authentication_errors`
- Description: "Test cases for authentication and authorization error responses"

✅ **Include test cases for all required scenarios**
- **Missing Credentials** (3 test cases):
  - AUTH-001: Missing API key in request headers (401 Unauthorized)
  - AUTH-002: Missing Bearer token in Authorization header (401 Unauthorized)
  - AUTH-003: Missing Basic auth credentials (401 Unauthorized)

- **Invalid Credentials** (3 test cases):
  - AUTH-004: Invalid API key format (403 Forbidden)
  - AUTH-005: Invalid Bearer token format (401 Unauthorized)
  - AUTH-006: Revoked API key (403 Forbidden)

- **Expired Credentials** (2 test cases):
  - AUTH-007: Expired Bearer token (401 Unauthorized)
  - AUTH-008: Expired API key (403 Forbidden)

- **Wrong Password** (2 test cases):
  - AUTH-009: Wrong password for basic authentication (401 Unauthorized)
  - AUTH-010: Wrong password with account lockout warning (401 Unauthorized)

- **Additional Coverage** (5 test cases):
  - AUTH-011 to AUTH-012: RBAC/Insufficient permissions (403 Forbidden)
  - AUTH-013 to AUTH-015: Happy path valid credentials (200 OK)

✅ **Each test case has clear input and expected error**
- All test cases include:
  - `id`: Unique identifier (e.g., AUTH-001)
  - `description`: Clear description of what is tested
  - `input_data`: Structured input dictionary with endpoint, headers, etc.
  - `expected_status`: HTTP status code (401, 403, 200)
  - `expected_error`: Error type (unauthorized, forbidden, or None)
  - `expected_message`: Expected error message substring
  - `tags`: Category tags for filtering

✅ **Follow the pattern defined in the base structure**
- Uses `TestCase` dataclass from `tests/test_tables.py`
- Uses `ErrorTestTable` for grouping related test cases
- Follows naming conventions (AUTH-XXX, lowercase tags)
- Implements tag-based filtering and categorization
- Supports metadata and custom validators

✅ **Test cases are runnable**
- Created `tests/test_auth_error_scenarios.py` - comprehensive pytest test suite
- Created `tests/runnable_auth_error_tests.py` - standalone test runner
- All 10 authentication error tests pass successfully
- Category-based filtering works: `--category missing-credentials`, `--category invalid-credentials`, etc.
- Execution time: < 0.1ms per test
