# Bead bf-64826u: Error Response Structure Validation Helper

## Summary

Fixed the export of error response structure validation helper functions from the test utils module. The helper functions themselves were already implemented in `tests/test_error_response_validation.py` with comprehensive test coverage in `tests/test_error_response_validation_cases.py`, but they were not properly exported from `tests/__init__.py`.

## Changes Made

### 1. Updated `tests/__init__.py`
- Added exports for all error response validation helper functions
- Added exports for related exception classes and pre-configured specs
- Now properly exports both HTTP validation helpers (status code, content-type) and error response structure validation helpers

### 2. Fixed Test Bug
- Fixed `test_multiple_custom_validators` in `tests/test_error_response_validation_cases.py`
- The test was using a `request_id` field that wasn't included in the spec's optional fields
- Updated to use a custom spec that includes `request_id` in optional fields

## Functionality

The helper functions support:

### Core Validation
- `validate_error_response()` - Generic validation with custom specs
- `validate_standard_error_response()` - Validates error + message fields
- `validate_detailed_error_response()` - Validates error + message + code fields
- `validate_error_field_only()` - Minimal validation for error field only

### Advanced Features
- Custom field validators (e.g., validate error codes are HTTP 4xx/5xx)
- JSON string parsing (accepts both dict and JSON strings)
- Multiple response validation (batch validation)
- HTTP response object support (extracts body automatically)
- Flexible specifications (required + optional field sets)

### Pre-configured Specs
- `STANDARD_ERROR_SPEC` - error + message (required), code + details (optional)
- `DETAILED_ERROR_SPEC` - error + message + code (required), details + stack_trace + request_id (optional)
- `MINIMAL_ERROR_SPEC` - error (required), message + code (optional)

### Built-in Validators
- `validate_error_code_is_int` - Ensures error code is integer
- `validate_error_code_is_http` - Ensures error code is 4xx or 5xx
- `validate_message_not_empty` - Ensures message is non-empty string
- `validate_error_not_empty` - Ensures error field is non-empty string

## Usage Example

```python
from tests import (
    validate_standard_error_response,
    validate_detailed_error_response,
    validate_error_response,
    ErrorResponseSpec,
    STANDARD_ERROR_SPEC,
)

# Standard validation (error + message)
response = {"error": "not_found", "message": "Resource not found"}
validate_standard_error_response(response)

# Detailed validation (error + message + code)
response = {"error": "auth_failed", "message": "Invalid credentials", "code": 401}
validate_detailed_error_response(response)

# Custom validation
custom_spec = ErrorResponseSpec(
    required_fields={"error", "message"},
    optional_fields={"code", "details", "request_id"}
)
response = {"error": "rate_limit", "message": "Too many requests", "code": 429}
validate_error_response(response, custom_spec)
```
